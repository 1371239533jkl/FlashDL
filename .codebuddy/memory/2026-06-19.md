# 2026-06-19

## UI 主题切换延迟问题修复

**问题**: 切换标签页和主题时出现严重 UI 延迟（部分变浅色/部分深色、图标不更新、tab 高亮延迟）

**根因**:
1. `QApplication.setStyleSheet()` 后 Qt 不会立即重绘，等待事件循环空闲
2. `QGraphicsDropShadowEffect` 极度消耗渲染性能，加剧延迟
3. 多处内联 `setStyleSheet()` 覆盖全局 QSS，主题切换时不更新
4. 历史卡片图标在创建时取色，主题切换后不会重建

**修复**:
- `main_window.py`: `_toggle_theme()` 增加 `app.processEvents()` + `self.repaint()` + `self.history_tab.refresh()`
- `main_window.py`: 移除 `QGraphicsDropShadowEffect`
- `styles.py`: 新增 `#BoldLabel`, `#EmptyLabel`, `#PlaylistHeader`, `#PlayerCtrlBtn` 全局 QSS 规则
- `download_tab.py`: `TaskCard.name_label` 改用 `setObjectName('BoldLabel')` 替换内联样式
- `history_tab.py`: `name_label` 改用 `setObjectName('BoldLabel')`, `empty_label` 改用 `setObjectName('EmptyLabel')`
- `player_tab.py`: 播放列表标题改用 `setObjectName('PlaylistHeader')`, 控制按钮改用 `setObjectName('PlayerCtrlBtn')`
